Multi-Client Sync

Core Feature

Sync your MCP servers to Claude Desktop, Cursor, Windsurf, VS Code, Claude Code, Codex, Gemini CLI and more with a single command. No more manually editing JSON config files scattered across different locations for each client. MCPSM automatically detects installed clients and updates their configurations.

Sync Commands
mcpsm clients list # See all detected clients
mcpsm clients sync # Sync to all enabled clients
mcpsm clients sync --client cursor # Sync to specific client
mcpsm clients enable claude # Enable a client for sync
mcpsm clients open cursor # Open client's config file

Interactive TUI

User Experience

Beautiful terminal interface with keyboard shortcuts. Navigate with arrow keys, toggle servers with Space, and perform actions without memorizing commands. The TUI provides real-time status updates, server health indicators, and a visual overview of your entire MCP setup. Perfect for users who prefer visual management over CLI commands.

Powerful CLI

Automation

Scriptable commands with JSON output for automation and CI/CD pipelines. Every TUI action has a CLI equivalent, ensuring 100% feature parity. Perfect for shell scripts, automation workflows, and quick terminal operations. Pipe JSON output to jq or other tools for advanced processing.

CLI Examples
mcpsm list # List all servers
mcpsm list --json # JSON output for scripting
mcpsm add my-server # Add server interactively
mcpsm enable my-server # Enable a server
mcpsm test --json | jq '.results' # Parse results

Parallel Testing

Reliability

Test all your servers simultaneously with parallel execution. See which ones are working, failing, or timing out before you use them in your AI client. Get detailed error messages, connection times, and tool counts for debugging. The test command spawns each server process and validates the MCP handshake.

Testing Commands
mcpsm test # Test all enabled servers
mcpsm test my-server # Test specific server
mcpsm test --timeout 30000 # Custom timeout (ms)
mcpsm test --json # JSON output with details
mcpsm doctor # Full health check

Tool Discovery

Visibility

Automatically discover all tools from your MCP servers. See tool names, descriptions, input schemas, and token costs. Know exactly what capabilities each server provides before enabling it. The discovery process connects to each server and retrieves the complete tool manifest with parameter definitions.

Discovery Commands
mcpsm tools list # List all discovered tools
mcpsm tools discover # Refresh tool discovery
mcpsm tools discover my-server # Discover from specific server
mcpsm tools list --json # Full tool schemas as JSON
mcpsm list --tokens # Show token counts per server

Token Counting

Cost Control

Track context usage per server and tool. Know exactly how many tokens your MCP tools consume in the system prompt. Optimize your setup by identifying heavy tools that eat into your context window. Token counts are calculated based on tool definitions including names, descriptions, and parameter schemas.

Token Commands
mcpsm tokens # Summary of token usage
mcpsm tokens --detailed # Per-tool breakdown
mcpsm tokens --json # JSON format
mcpsm list --tokens # Token counts in server list
# TUI shows tokens next to each server: "11 tools · 4.2k tokens"

Tool Filtering

Optimization

Enable or disable specific tools per server. Reduce token usage by hiding tools you don't need. Keep your AI context lean and focused on the tools that matter for your workflow. Disabled tools are removed from the system prompt, freeing up context space without removing the entire server.

Filtering Commands
mcpsm tools list my-server # List tools for server
mcpsm tools disable my-server/tool-name # Disable tool
mcpsm tools enable my-server/tool-name # Enable tool
mcpsm tools reset my-server # Reset to all enabled
# In TUI: Press T to open Tools screen, Space to toggle

Profiles

Organization

Group servers by project or context. Create profiles for work, personal, or project-specific setups. Switch between configurations instantly without manually enabling/disabling servers. Each profile remembers which servers are enabled, so you can have different tool sets for different tasks.

Profile Commands
mcpsm profile list # List all profiles
mcpsm profile create work # Create new profile
mcpsm profile use work # Switch to profile
mcpsm profile add work my-server # Add server to profile
mcpsm profile delete old-profile # Delete profile

Import & Export

Migration

Migrate configs between machines or import servers from existing client configurations. Already have servers configured in Claude Desktop or Cursor? Import them directly. Export your MCPSM setup to share with teammates, backup your configuration, or migrate to a new machine.

Import/Export Commands
mcpsm import --from claude # Import from Claude Desktop
mcpsm import --from cursor # Import from Cursor
mcpsm import --file backup.json # Import from file
mcpsm export > backup.json # Export all servers
mcpsm export --format claude # Export in Claude format

Remote Servers

Connectivity

Connect to HTTP and SSE remote MCP servers. Full support for bearer tokens, custom headers, and various authentication methods. Access cloud-hosted MCP services, team-shared servers, or your own deployed MCP endpoints. Remote servers support both HTTP transport and Server-Sent Events (SSE) for real-time streaming.

Remote Server Commands
mcpsm add --type http # Add HTTP remote server
mcpsm add --type sse # Add SSE remote server
# During add, you can specify:
# - URL: https://api.example.com/mcp
# - Bearer token for authentication
# - Custom headers (key: value pairs)

OAuth Support

Security

Built-in OAuth 2.0 flow with PKCE for remote servers that require authentication. Securely authenticate with enterprise MCP services without exposing credentials. The OAuth flow opens your browser for authentication and securely stores refresh tokens for automatic token renewal.

OAuth Commands
mcpsm auth login my-remote-server # Start OAuth flow
mcpsm auth logout my-remote-server # Revoke tokens
mcpsm auth status # Check auth status
mcpsm auth refresh my-remote-server # Refresh tokens
# Tokens are stored securely in ~/.mcp-manager/auth/

Daemon Mode

Background

Run the MCP gateway in background with auto-start on system boot. Set it up once and forget it. Your servers are always available when you need them. The daemon manages server processes, handles restarts on failure, and provides a unified gateway endpoint for all your MCP servers.

Daemon Commands
mcpsm daemon start # Start the daemon
mcpsm daemon stop # Stop the daemon
mcpsm daemon status # Check daemon status
mcpsm daemon startup enable # Auto-start on boot
mcpsm daemon logs --follow # View daemon logs
