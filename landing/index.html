Multi-Client Sync

Core

Sync your MCP servers to Claude Desktop, Cursor, Windsurf, VS Code, Claude Code, Kiro, Codex, Gemini CLI, Zed and more. No more manually editing JSON config files.

Client Commands
mcpsm clients list # See all detected clients
mcpsm clients connect claude # Connect a client
mcpsm clients connect cursor # Connect specific client
mcpsm clients disconnect windsurf # Disconnect a client
mcpsm clients open cursor # Open client's config file

Parallel Testing

Reliability

Test all your servers simultaneously with parallel execution. See which ones are working, failing, or timing out before you use them in your AI client.

Testing Commands
mcpsm test # Test all servers
mcpsm test my-server # Test specific server
mcpsm test --json # JSON output with details
mcpsm doctor # Full health check

Tool Discovery

Visibility

Automatically discover all tools from your MCP servers. See tool names, descriptions, input schemas, and token costs before enabling.

Discovery Commands
mcpsm tools list # List all discovered tools
mcpsm tools discover my-server # Discover tools from server
mcpsm tools list --json # Full tool schemas as JSON
mcpsm list --tokens # Show token counts per server

Token Counting

Cost Control

Track context usage per server and tool. Know exactly how many tokens your MCP tools consume in the system prompt. Optimize by identifying heavy tools.

Token Commands
mcpsm tokens # Summary of token usage
mcpsm tokens --detailed # Per-tool breakdown
mcpsm tokens --json # JSON format
mcpsm list --tokens # Token counts in server list

Tool Filtering

Optimization

Enable or disable specific tools per server. Reduce token usage by hiding tools you don't need. Keep your AI context lean and focused.

Filtering Commands
mcpsm tools list my-server # List tools for server
mcpsm tools disable my-server tool-name # Disable a tool
mcpsm tools enable my-server tool-name # Enable a tool
mcpsm tools enable my-server # Enable all tools

Profiles

Organization

Group servers by project or context. Create profiles for work, personal, or project-specific setups. Switch between configurations instantly.

Profile Commands
mcpsm profile list # List all profiles
mcpsm profile create work # Create new profile
mcpsm profile use work # Switch to profile
mcpsm profile add work my-server # Add server to profile
mcpsm profile delete old-profile # Delete profile

Import & Export

Migration

Migrate configs between machines or import servers from existing client configurations. Already have servers in Claude Desktop or Cursor? Import them directly.

Import/Export Commands
mcpsm import --from claude # Import from Claude Desktop
mcpsm import --from cursor # Import from Cursor
mcpsm import backup.json # Import from file
mcpsm export --output backup.json # Export all servers
mcpsm export --format claude # Export in Claude format

Remote Servers

Connectivity

Connect to HTTP and SSE remote MCP servers. Full support for bearer tokens, custom headers, and various authentication methods.

Remote Server Commands
mcpsm add --type http # Add HTTP remote server
mcpsm add --type sse # Add SSE remote server
# During add, you can specify:
# - URL: https://api.example.com/mcp
# - Bearer token for authentication
# - Custom headers (key: value pairs)

OAuth Support

Security

Built-in OAuth 2.0 flow with PKCE for remote servers that require authentication. Securely authenticate with enterprise MCP services.

OAuth Commands
mcpsm auth login my-remote-server # Start OAuth flow
mcpsm auth logout my-remote-server # Revoke tokens
mcpsm auth status # Check auth status
mcpsm auth refresh my-remote-server # Refresh tokens

Daemon Mode

Background

Run the MCP gateway in background with auto-start on system boot. Set it up once and forget it. Your servers are always available.

Daemon Commands
mcpsm daemon start # Start the daemon
mcpsm daemon stop # Stop the daemon
mcpsm daemon status # Check daemon status
mcpsm daemon startup enable # Auto-start on boot
mcpsm daemon logs --follow # View daemon logs
